# 💾 备份与恢复

## 备份机制

McGuffin 提供两层备份保护：

1. **自动备份** — 定时自动创建备份
2. **手动备份** — 管理员随时触发

数据存储：完整备份（SQLite + 内存数据序列化）

## 自动备份

通过 `config.toml` 配置自动备份：

```toml
[backup]
interval_minutes = 60    # 每 60 分钟备份一次
retention_count = 48     # 保留最近 48 个备份
```

- 启动时自动启动定时备份线程
- 备份文件存于 `backups/` 目录
- 超出保留数的旧备份自动删除

## 手动操作

### 管理后台

在 `/admin/backups` 页面：

**创建备份**：
1. 点击「创建备份」
2. 等待创建完成（通常 < 5 秒）
3. 备份出现在列表中

**查看备份列表**：
- 文件名、大小、修改时间
- 每个备份可执行：下载、恢复、删除

**恢复备份**：
1. 点击目标备份的「恢复」
2. 确认操作（系统会自动创建 `pre_restore_` 前缀的安全快照）
3. 恢复完成后刷新页面

**下载备份**：
1. 点击「下载」
2. 保存为本地文件

**删除备份**：
1. 点击「删除」
2. 确认操作

### 导入/导出

**导出数据（JSON 格式）**：
- 导出所有数据为 JSON 文件
- 适用于跨实例迁移

**导出配置（TOML 格式）**：
- 导出当前配置为 TOML 文件

**导入数据**：
1. 点击「导入数据 (.json / .db)」
2. 选择备份文件
3. 确认替换所有现有数据

**导入配置**：
1. 点击导入 TOML 配置
2. 确认操作

## CLI 操作

```bash
# 创建备份
mcguffin backup create

# 列表示例
mcguffin backup list
# → backups/
#   ├── mcguffin-backup-2026-07-14_143000.json
#   ├── mcguffin-backup-2026-07-14_144000.json
#   └── mcguffin-backup-2026-07-14_145000.json

# 恢复指定备份
mcguffin backup restore mcguffin-backup-2026-07-14_143000.json

# 删除备份
mcguffin backup delete mcguffin-backup-2026-07-14_143000.json
```

## 备份文件格式

备份采用 JSON 格式，包含以下数据：

```json
{
  "version": "0.3.0",
  "created_at": "2026-07-14T14:30:00Z",
  "users": { ... },
  "sessions": { ... },
  "problems": { ... },
  "contests": { ... },
  "posts": { ... },
  "teams": { ... },
  "join_requests": { ... },
  "notifications": { ... },
  "member_groups": { ... },
  ...
}
```

## 最佳实践

1. **设置合理的自动备份间隔**：生产环境推荐 30-60 分钟
2. **保留足够的备份**：至少保留 24 小时内的备份
3. **定期下载备份到外部存储**：不要依赖单一存储
4. **恢复前注意**：恢复操作会替换所有现有数据
5. **大版本升级前**：先手动创建备份
