# 💬 社区讨论

## 帖子系统

McGuffin 使用统一的帖子系统，涵盖讨论、公告、建议等多种类型。

## 浏览帖子

1. 进入「社区」页面
2. 帖子列表显示：标题、作者、标签、发布时间、回复数、表情反应数
3. 可点击进入帖子详情查看完整内容和回复

## 创建帖子

需要 `view_discussions` 权限。

1. 在社区页面点击「发布」
2. 选择帖子类型（可选，影响 emoji 图标）：
   - 💬 讨论
   - 📢 公告（需 `manage_posts` 权限）
   - 💡 建议
3. 填写标题和内容（Markdown 格式）
4. 添加标签（可选）
5. 设置选项：
   - **仅团队成员可见**（可选）
   - **置顶**（需 `manage_posts` 权限）
6. 发布

## 回复与互动

**回复帖子**：
1. 进入帖子详情
2. 在底部输入框编写回复
3. 支持 Markdown 格式
4. 可 @ 提及用户（`@用户名`）

**表情反应**：
1. 在帖子或回复上点击反应按钮
2. 选择 emoji
3. 再次点击取消反应

**编辑/删除**：
- 作者可编辑和删除自己的帖子/回复
- 管理员和管理员可编辑和删除任何帖子/回复
- 删除回复后，回复内容不可恢复

## 标签系统

帖子可附加标签（如 #求助、#分享、#公告），便于分类和筛选。

**管理标签**（需 `manage_tags` 权限）：
- 可在管理后台配置可用标签列表
- 标签 emoji 可自定义

## 社区动态

「社区」首页展示公开的帖子流（`/community`）：

- 显示所有公开帖子
- 按时间排序
- 包含回复计数和反应统计
- 支持通过标签筛选

## 建议工作流

建议类帖子有一个额外的状态字段：

| 状态 | 说明 |
|------|------|
| `open` | 建议已提交，等待处理 |
| `accepted` | 已被采纳 |
| `rejected` | 未被采纳 |
| `implemented` | 已实现 |

管理员可在建议详情页中变更状态。

## 置顶逻辑

- 置顶帖子始终显示在列表顶部
- 多个置顶帖子按时间排序
- 普通帖子按最后活动时间排序

## 可见性

| 帖子设置 | 可见范围 |
|----------|----------|
| 公开（`team_only=false`） | 所有人 |
| 仅团队成员（`team_only=true`） | 已加入团队的成员 |
| ACL 限制 | 指定的用户/组 |
